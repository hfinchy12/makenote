  <action>
Create `src/makenote/cli.py`:

```python
import sys
import click
from makenote.config import config_exists, run_config_flow


@click.group(invoke_without_command=True)
@click.version_option(version="0.1.0", prog_name="mn", message="%(prog)s %(version)s")
@click.pass_context
def main(ctx: click.Context) -> None:
    """mn — fast terminal note logging."""
    if not config_exists() and ctx.invoked_subcommand != "config":
        click.echo("No config found. Running first-time setup.")
        run_config_flow()
    elif ctx.invoked_subcommand is None:
        click.echo(ctx.get_help())


@main.command()
def config() -> None:
    """Edit configuration."""
    run_config_flow()
```

Critical constraints from research:
- `message="%(prog)s %(version)s"` is REQUIRED — the click default includes a comma and the word "version", which breaks UX-01.
- `ctx.invoked_subcommand != "config"` guard is REQUIRED — without it, `mn config` triggers the auto-setup AND the config subcommand (Pitfall 3 in research).
- Docstrings must be short (one line max) — click includes them verbatim in --help output. Long docstrings violate the "minimal output" locked decision.
- Use `click.echo()` for ALL output — never `print()`.

Note: `config_exists` and `run_config_flow` are imported from `makenote.config` which does not exist until Plan 03. The xfail test stubs from Plan 01 accommodate this. Run tests after Plan 03 completes to see them turn green.
