    - write_note(repo, subject, note_text) builds path as f"notes/{subject}/notes.jsonl"; GETs current file; appends new JSONL record; PUTs back with sha if file existed
    - read_notes(repo, subjects) fetches each subject file; decodes base64; parses JSONL lines; returns list sorted newest-first, max 20 records

  </behavior>
  <action>
    Implement src/makenote/github.py following the architecture from RESEARCH.md Pattern 1-4 exactly.

    Module-level constant: `NOTES_ROOT = "notes"`

    Error hierarchy:
    ```
    class GhError(Exception): pass
    class GhNotInstalledError(GhError): pass
    class GhNotAuthError(GhError): pass
    class ShaConflictError(GhError): pass
    class GhApiError(GhError): pass
    ```

    _run_gh(*args): use shutil.which("gh") check first. On returncode != 0, check stderr.lower() for "not logged in" or "auth" or "authentication" to raise GhNotAuthError; otherwise raise GhApiError(stderr.strip()).

    write_note(repo, subject, note_text):
    1. path = f"{NOTES_ROOT}/{subject}/notes.jsonl"
    2. endpoint = f"repos/{repo}/contents/{path}"
    3. GET via subprocess.run(["gh", "api", endpoint], capture_output=True, text=True) — NOT _run_gh, because a 404 here is normal (file doesn't exist yet)
    4. Build new record: json.dumps({"date": datetime.date.today().isoformat(), "subject": subject, "note": note_text})
    5. If GET succeeded: decode existing content, append record, get sha
    6. base64-encode new content
    7. PUT: if existing_sha include sha field; if PUT returncode != 0, check for "422" or "conflict" in stderr → ShaConflictError; else GhApiError

    read_notes(repo, subjects): for each subject, GET file via subprocess.run (404 = skip silently); decode; parse each JSONL line; return sorted by "date" descending, max 20.

    After writing, run pytest to reach GREEN: `pytest tests/test_github.py -x -q`
