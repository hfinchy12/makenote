## Tech Debt

### Phase 01-scaffold-and-config

- **Dead code `constants.py`** — `CONFIG_PATH` is defined here but `config.py` maintains its own independent copy. The file is imported by nothing. Risk: future callers could import the wrong path. Remediation: either delete `constants.py` or change it to `from makenote.config import CONFIG_PATH`.

### Phase 02-core-note-taking

- **Empty `default_subject` edge case** — `run_config_flow()` allows saving config with `default_subject = ""`. When `mn d` is run with an empty default, the GitHub path becomes `notes//notes.jsonl` (double slash). Not a crash. Remediation: add validation in `run_config_flow()` or in `default_note()` before calling `write_note()`.

### Phase 03-distribution

- **PLACEHOLDER formula hashes** — `Formula/makenote.rb` sha256 fields and resource stanza URLs are intentional placeholders. `brew install` will fail until the release workflow in `RELEASING.md` is run after tagging v0.1.0. Documented and expected.

### Total: 3 items across 2 phases (1 actionable, 1 latent edge case, 1 planned release step)
